Self Troubleshooting Guide

Please follow the troubleshooting steps below and see if any of the following have fixed your technical issue. If the issue persists after completing these steps, contact the COLTT Help Desk and include the device type, browser, and exact error message.

Step 1. Try a recommended browser

It is recommended to access Brightspace using Google Chrome or Mozilla Firefox.

If you prefer to use a different browser, ensure it is up to date and that you conduct maintenance on it regularly.

  • Safari: Apple Support
  • Microsoft Edge: Microsoft Support
  • Opera: Opera Support
Step 2. Enable third-party cookies

Some Brightspace, Panopto, and SoftChalk tools require third-party cookies to function correctly. Same with external tools.

  • Chrome support: Third-Party Cookies: Chrome Support | Computer / Android / iPhone & iPad
  • Mac (Chrome): Chrome App → three dots → Settings → Privacy and Security → Cookies and other site data → turn Block third-party cookies off.
  • Firefox: Enable and disable cookies
Step 3. Clear browsing data

In cases where tools are not functioning correctly, clearing your browser's cache and cookies may resolve the issue. Once you have cleared the data, try accessing the tool once more; you may need to log in again.

  • Chrome support: Clear cache & cookies: Chrome Support | Computer / Android / iPhone & iPad
  • Mac (Chrome): Follow the same Chrome steps above through Settings.
  • Firefox: Clear cookies and site data
Step 4. Check cross-site tracking settings on iPad

On iPad, third-party cookies may be disabled by default. To enable them:

Settings → Chrome → Prevent Cross-Website Tracking (turn this option off).

Make sure this setting is configured correctly for the tool you are using. It should be disabled if the tool requires third-party cookies.

Step 5. Check for browser updates

If it seems like your browser is outdated, updating it may resolve the issue.

  • Chrome: Chrome App → three dots → Help → About Google Chrome.
  • Chromebook: This process may differ depending on ChromeOS version. Please see Step 6.
  • iPad (Chrome): Open App Store → Profile → Available Updates → search for Chrome → tap Update.
  • Firefox: Go to Firefox → Help → About Firefox
Step 6. Check for device updates

Keeping your device's operating system up to date is important for security and performance. If your device is outdated, it may struggle to run certain applications or access certain websites.

  • Windows: Windows logo → Settings → Windows Update → Check for Updates
  • iPad: Settings → General → Software Update → enable Automatic Updates if preferred
  • Mac: Apple menu → System Settings → General → Software Update
  • Chromebook: Time (bottom right) → Settings → About ChromeOS → Check for updates → Restart if prompted
Step 7. Other common fixes

If the issue persists after completing the previous steps, ensure the following:

  • You are connected to a valid Wi-Fi network and not using a hotspot.
  • No VPNs are active.
  • Disable any Do Not Track options.
  • Reset your password if the issue may be related to account access.
  • Remove any browser extensions that might be causing conflicts.
  • Try accessing the tool on a different device to see if the issue is specific to one device.
  • Disable any ad blockers or privacy tools that might be interfering.
  • Disable any virus protection software that might be blocking access.
  • Disable or remove any virtual machines or containers that might be causing conflicts. Including game mods.
When is it an IT issue?
  • The device is outdated or no longer usable. Usually devices older than 2-3 years; recommend replacement if older than 5 years.
  • An administrator needs to change protected settings on a UTRGV-managed device.
  • The issue is linked to the student’s myUTRGV account.
  • The student is incoming or returning and needs their account unlocked.
  • The issue involves IT-managed software or myUTRGV apps.

See: Software Support Articles

If the issue continues after completing these steps, contact the COLTT support team and include the device type, browser, and exact error message.
