HTML Templates

These are “free content widgets” you can add to a Brightspace course by creating an HTML Document in the Content area and pasting the code.

Each template includes instructions and a preview of how it will look once pasted into Brightspace. You can customize the content, but try to keep the structure and styles for accessibility and consistency.

Please note that some templates may require additional setup or customization to work properly in Brightspace.

How to Use in Brightspace

Use an HTML Template (Content Area)

  1. Go to Content in your course.
  2. Select Create New → HTML Document.
  3. Open Source (the </> icon).
  4. Copy the template HTML from this site and paste it into the </> editor.
  5. Replace placeholders (titles, dates, links, etc.) with your course content.
  6. Save and preview on desktop + mobile.

Tip: Keep headings in order (H1 → H2 → H3) and avoid relying on color alone to convey meaning.

Best Accessibility Practices

Creating accessible content ensures that all students, including those using screen readers, keyboard navigation, or mobile devices, can successfully engage with your course. These practices are simple to apply and improve clarity for everyone.

1. Use Clear Structure with Headings

Organize your content using headings (Heading 1, Heading 2, Heading 3). This helps students quickly scan your content and allows screen readers to navigate sections easily.

  • Use headings in order (H1 → H2 → H3)
  • Do not skip levels (for example, don’t jump from H1 to H3)
  • Avoid bold text as a replacement for headings

Why it matters: Students using assistive technology rely on headings like a table of contents.

2. Add Alternative Text (Alt Text) to Images

Alt text is a short description of an image. It is read aloud by screen readers.

  • Describe the purpose of the image, not every detail
  • If the image is decorative, mark it as decorative (no description needed)

Example: Instead of “image123.jpg,” write “Bar chart showing student grades increasing over time.”

Why it matters: Students who cannot see the image still understand the content.

3. Use High Color Contrast

Make sure text is easy to read against its background.

  • Use dark text on a light background (or vice versa)
  • Avoid light gray text or bright colors on white
  • Do not rely on color alone to convey meaning

Example: Instead of “items in red are important,” write “important items are marked in red and labeled ‘Required.’”

Why it matters: Students with visual impairments or color blindness need clear contrast.

4. Write Descriptive Links

Links should clearly describe where they go.

  • Avoid: “Click here”
  • Use: “View the Week 2 Study Guide”

Why it matters: Screen readers often list links separately—students need context.

5. Use Lists Properly

Use built-in list tools instead of typing dashes or numbers manually.

  • Use bullet lists for general items
  • Use numbered lists for steps or sequences

Why it matters: Screen readers announce lists and help students follow content more easily.

6. Keep Tables Simple and Meaningful

Tables should only be used for data, not layout.

  • Add column or row headers
  • Avoid merged or complex cells
  • Include a brief description if needed

Why it matters: Screen readers read tables cell-by-cell using headers for context.

7. Avoid Uploading Unchecked PDFs

Not all PDFs are accessible by default.

  • Use HTML pages in Brightspace when possible
  • If using PDFs, ensure they are tagged and readable

Why it matters: Many PDFs cannot be read properly by assistive technologies.

8. Write Clearly and Simply

Use straightforward language and break up large blocks of text.

  • Use short paragraphs
  • Use headings and spacing
  • Avoid overly complex sentences

Why it matters: Clear writing helps all students, including those with cognitive disabilities.

Final Tip

Use the Brightspace Accessibility Checker while editing your content. It will guide you through fixing common issues step-by-step.
